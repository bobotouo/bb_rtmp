# RTMP 推流示例应用

这是一个完整的 RTMP 推流测试应用，展示了如何使用 `bb_rtmp` 插件进行推流。

## 功能特性

- ✅ RTMP 地址输入和初始化
- ✅ 摄像头预览
- ✅ 开始/停止推流
- ✅ 前后摄像头切换
- ✅ 分辨率切换（360p, 480p, 720p, 1080p）
- ✅ 码率调整（1-4 Mbps）
- ✅ 实时状态显示
- ✅ 错误提示

## 使用方法

1. **配置 RTMP 地址**
   - 在输入框中输入你的 RTMP 推流地址
   - 格式：`rtmp://your-server.com/live/stream`

2. **初始化**
   - 点击"初始化"按钮
   - 等待摄像头预览出现

3. **开始推流**
   - 点击"开始推流"按钮
   - 状态栏会显示推流状态

4. **测试功能**
   - 切换摄像头：测试前后摄像头切换
   - 切换分辨率：测试不同分辨率
   - 调整码率：测试码率控制
   - 查看状态：点击右上角信息图标刷新状态

5. **停止和释放**
   - 点击"停止推流"停止推流
   - 点击"释放资源"释放所有资源

## 注意事项

- 切换分辨率需要先停止推流
- 确保设备有摄像头和麦克风权限
- RTMP 服务器需要支持 H.264 视频和 AAC 音频
- 建议在真实设备上测试，模拟器可能不支持硬件编码

## 测试建议

1. **基础测试**
   - 使用本地 RTMP 服务器（如 nginx-rtmp）进行测试
   - 验证视频和音频是否正常推流

2. **功能测试**
   - 测试摄像头切换
   - 测试分辨率切换
   - 测试码率调整
   - 测试自适应码率（网络状况变化时）

3. **性能测试**
   - 观察 CPU 和内存使用
   - 检查推流延迟
   - 验证零拷贝是否生效

4. **稳定性测试**
   - 长时间推流测试
   - 网络中断恢复测试
   - 前后台切换测试
